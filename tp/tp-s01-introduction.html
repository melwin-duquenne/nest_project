TP 01 — Initialisation du projet TaskFlow API

Durée : 30 minutes Objectif : créer le projet NestJS, explorer sa structure, ajouter une route /health.

Pré-requis

Vérifier l'environnement avant de commencer :

node --version    # doit afficher v20.x ou supérieur
npm --version     # doit afficher 10.x ou supérieur
docker --version  # Docker Engine ou Docker Desktop
nest --version    # @nestjs/cli global

Si nest n'est pas disponible :

npm install -g @nestjs/cli

Étape 1 — Créer le projet

Dans votre dossier de travail :

nest new taskflow-api
  • Choisir npm comme gestionnaire de paquets
  • Attendre l'installation des dépendances
cd taskflow-api

Étape 2 — Explorer la structure générée

Ouvrir le projet dans VS Code :

code .

Observer l'arborescence :

taskflow-api/
├── src/
│   ├── app.controller.ts    ← contrôleur racine
│   ├── app.controller.spec.ts
│   ├── app.module.ts        ← module racine
│   ├── app.service.ts       ← service racine
│   └── main.ts              ← point d'entrée
├── test/
├── nest-cli.json
├── package.json
└── tsconfig.json

Questions à se poser :

  • Que fait main.ts ? Comment l'application démarre-t-elle ?
  • Comment AppModule est-il lié à AppController et AppService ?
  • Que retourne la route GET / par défaut ?

Démarrer l'application en mode développement :

npm run start:dev

Tester avec curl :

curl http://localhost:3000
# → Hello World!

Étape 3 — Modifier la route par défaut

Modifier src/app.service.ts pour que getHello() retourne un objet JSON :

{
  message: 'TaskFlow API',
  version: '1.0.0',
}

Pensez à mettre à jour le type de retour dans AppController en conséquence.

Tester :

curl http://localhost:3000
# → {"message":"TaskFlow API","version":"1.0.0"}

Étape 4 — Ajouter la route /health

Dans AppController, ajouter une méthode qui répond sur GET /health et retourne :

{
  status: 'ok',
  timestamp: new Date().toISOString(),
}

Indice : regardez le décorateur @Get() déjà présent — il accepte un chemin en paramètre.

Tester :

curl http://localhost:3000/health
# → {"status":"ok","timestamp":"2026-03-05T..."}

Étape 5 — Fichier de test HTTP

Créer src/http/app.http (VS Code REST Client) :

### Racine
GET http://localhost:3000

### Health check
GET http://localhost:3000/health

Cliquer sur "Send Request" au-dessus de chaque requête pour tester.

Checkpoint ✓

  • node --version ≥ 20
  • npm run start:dev démarre sans erreur
  • GET / retourne {"message":"TaskFlow API","version":"1.0.0"}
  • GET /health retourne {"status":"ok","timestamp":"..."}
  • Fichier src/http/app.http créé

Fil rouge après S01

Le repo taskflow-api est créé et la route /health répond 200.
