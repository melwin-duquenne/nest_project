TP 16 — Prisma ORM : Introduction & Migration

Durée : 45 minutes Objectif : configurer Prisma, définir le schéma TaskFlow, migrer, et réécrire UsersService avec Prisma Client.

Note : on travaille sur une branche séparée pour ne pas casser le projet TypeORM.

Étape 1 — Branche et installation

git checkout -b feature/prisma
npm install prisma @prisma/client
npx prisma init

Étape 2 — Configurer la base de données Prisma

Dans .env, ajouter :

DATABASE_URL="postgresql://taskflow:taskflow@localhost:5432/taskflow_prisma?schema=public"

Créer la base :

docker exec -it taskflow_postgres psql -U taskflow -c "CREATE DATABASE taskflow_prisma;"

Étape 3 — Définir le schéma Prisma

Remplacer le contenu de prisma/schema.prisma avec le schéma complet de TaskFlow :

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

enum UserRole     { admin member viewer }
enum ProjectStatus { active archived draft }
enum TaskStatus    { todo in_progress done }
enum TaskPriority  { low medium high }

model User {
  id           String   @id @default(uuid())
  email        String   @unique
  passwordHash String   @map("password_hash")
  name         String
  role         UserRole @default(member)
  createdAt    DateTime @default(now()) @map("created_at")
  updatedAt    DateTime @updatedAt @map("updated_at")

  teams    Team[]    @relation("TeamMembers")
  tasks    Task[]    @relation("TaskAssignee")
  comments Comment[]

  @@map("users")
}

model Team {
  id          String   @id @default(uuid())
  name        String   @unique
  description String?
  createdAt   DateTime @default(now()) @map("created_at")
  updatedAt   DateTime @updatedAt @map("updated_at")

  members  User[]    @relation("TeamMembers")
  projects Project[]

  @@map("teams")
}

model Project {
  id          String        @id @default(uuid())
  name        String
  description String?
  status      ProjectStatus @default(draft)
  teamId      String        @map("team_id")
  createdAt   DateTime      @default(now()) @map("created_at")
  updatedAt   DateTime      @updatedAt @map("updated_at")

  team  Team   @relation(fields: [teamId], references: [id], onDelete: Cascade)
  tasks Task[]

  @@map("projects")
}

model Task {
  id          String       @id @default(uuid())
  title       String
  description String?
  status      TaskStatus   @default(todo)
  priority    TaskPriority @default(medium)
  projectId   String       @map("project_id")
  assigneeId  String?      @map("assignee_id")
  createdAt   DateTime     @default(now()) @map("created_at")
  updatedAt   DateTime     @updatedAt @map("updated_at")

  project  Project   @relation(fields: [projectId], references: [id], onDelete: Cascade)
  assignee User?     @relation("TaskAssignee", fields: [assigneeId], references: [id], onDelete: SetNull)
  comments Comment[]

  @@map("tasks")
}

model Comment {
  id        String   @id @default(uuid())
  content   String
  authorId  String   @map("author_id")
  taskId    String   @map("task_id")
  createdAt DateTime @default(now()) @map("created_at")

  author User @relation(fields: [authorId], references: [id], onDelete: Cascade)
  task   Task @relation(fields: [taskId], references: [id], onDelete: Cascade)

  @@map("comments")
}

Lisez ce schéma attentivement. Repérez les différences de syntaxe avec TypeORM.

Étape 4 — Créer et appliquer la migration

npx prisma migrate dev --name init-taskflow

Explorer le schéma dans Prisma Studio :

npx prisma studio
# → http://localhost:5555

Étape 5 — PrismaModule + PrismaService

Ces fichiers sont de la configuration standard — fournis :

src/prisma/prisma.service.ts :

import { Injectable, OnModuleInit, OnModuleDestroy } from '@nestjs/common';
import { PrismaClient } from '@prisma/client';

@Injectable()
export class PrismaService extends PrismaClient
  implements OnModuleInit, OnModuleDestroy {
  async onModuleInit() {
    await this.$connect();
  }
  async onModuleDestroy() {
    await this.$disconnect();
  }
}

src/prisma/prisma.module.ts :

import { Global, Module } from '@nestjs/common';
import { PrismaService } from './prisma.service';

@Global()
@Module({
  providers: [PrismaService],
  exports: [PrismaService],
})
export class PrismaModule {}

Importer PrismaModule dans AppModule.

Étape 7 — Comparer TypeORM vs Prisma

Testez les deux implémentations sur les mêmes routes et observez :

  • La concision du code Prisma (select intégré vs { select: false } sur la colonne)
  • include pour les relations vs { relations: [...] }
  • L'absence de @InjectRepository et Repository<T>
