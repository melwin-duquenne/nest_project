Étape 4 — Controller Users

Mettre à jour src/users/users.controller.ts pour exposer les 5 routes CRUD :

Méthode Route Handler Status
GET /users findAll() 200
GET /users/:id findOne(id) 200
POST /users create(dto) 201
PATCH /users/:id update(id, dto) 200
DELETE /users/:id remove(id) 204

Décorateurs utiles : @Get(), @Post(), @Patch(), @Delete(), @Param(), @Body(), @HttpCode(HttpStatus.CREATED), @HttpCode(HttpStatus.NO_CONTENT)

Étape 5 — Tests avec REST Client

Créer src/http/users.http :

@baseUrl = http://localhost:3000/api

### Lister tous les utilisateurs
GET {{baseUrl}}/users

### Créer un utilisateur
POST {{baseUrl}}/users
Content-Type: application/json

{
  "email": "alice@taskflow.dev",
  "name": "Alice Dupont",
  "role": "admin"
}

### Créer un deuxième utilisateur
POST {{baseUrl}}/users
Content-Type: application/json

{
  "email": "bob@taskflow.dev",
  "name": "Bob Martin",
  "role": "member"
}

### Récupérer un utilisateur (remplacer l'ID)
GET {{baseUrl}}/users/REMPLACER_PAR_UN_ID

### Mettre à jour un utilisateur
PATCH {{baseUrl}}/users/REMPLACER_PAR_UN_ID
Content-Type: application/json

{
  "name": "Alice Dupont-Martin"
}

### Supprimer un utilisateur
DELETE {{baseUrl}}/users/REMPLACER_PAR_UN_ID

### Cas 404 — ID inexistant
GET {{baseUrl}}/users/00000000-0000-0000-0000-000000000000

### Cas 409 — Email déjà utilisé
POST {{baseUrl}}/users
Content-Type: application/json

{
  "email": "alice@taskflow.dev",
  "name": "Autre Alice"
}

Validation des scénarios

Tester tous les scénarios :

Scénario Route Réponse attendue
Lister (vide) GET /api/users [] (200)
Créer Alice POST /api/users User créé (201)
Créer Bob POST /api/users User créé (201)
Lister GET /api/users [alice, bob] (200)
Récupérer Alice GET /api/users/:id Alice (200)
Modifier Alice PATCH /api/users/:id Alice modifiée (200)
ID inexistant GET /api/users/bad-id 404 + message
Email dupliqué POST /api/users même email 409 + message
Supprimer Bob DELETE /api/users/:id (204 No Content)

Checkpoint ✓

  • GET /api/users → []
  • POST /api/users avec email + name → User créé avec id, createdAt
  • GET /api/users/:id → User (200) ou 404
  • PATCH /api/users/:id → User mis à jour
  • DELETE /api/users/:id → 204 No Content
  • Email dupliqué → 409
  • ID inexistant → 404 avec message clair
