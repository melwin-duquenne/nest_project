TP 12 — Swagger / OpenAPI

Durée : 45 minutes Objectif : documenter l'API TaskFlow avec Swagger, ajouter des exemples sur les DTOs, tester via l'UI.

Étape 1 — Installation

npm install @nestjs/swagger swagger-ui-express

Étape 2 — Configurer Swagger dans main.ts

Modifier src/main.ts pour initialiser Swagger en dehors de la production :

import { SwaggerModule, DocumentBuilder } from '@nestjs/swagger';

if (process.env.NODE_ENV !== 'production') {
  const config = new DocumentBuilder()
    .setTitle('TaskFlow API')
    .setDescription('API RESTful de gestion de projets et tâches')
    .setVersion('1.0')
    .addBearerAuth(
      {
        type: 'http',
        scheme: 'bearer',
        bearerFormat: 'JWT',
        description: 'Token JWT obtenu via POST /api/auth/login',
      },
      'JWT-auth',
    )
    .addTag('auth', 'Authentification')
    .addTag('users', 'Gestion des utilisateurs')
    .addTag('teams', 'Gestion des équipes')
    .addTag('projects', 'Gestion des projets')
    .addTag('tasks', 'Gestion des tâches')
    .addTag('comments', 'Commentaires')
    .build();

  const document = SwaggerModule.createDocument(app, config);
  SwaggerModule.setup('docs', app, document, {
    swaggerOptions: { persistAuthorization: true },
  });
}

Vérifier : http://localhost:3000/docs

Étape 3 — Annoter les controllers

Pour chaque controller, ajoutez les décorateurs Swagger appropriés.

Décorateurs disponibles dans @nestjs/swagger :

  • @ApiTags('nom') — groupe les routes dans l'UI
  • @ApiBearerAuth('JWT-auth') — indique que la route nécessite un token
  • @ApiOperation({ summary: '...' }) — description de la route
  • @ApiOkResponse(), @ApiCreatedResponse(), @ApiNoContentResponse() — succès
  • @ApiNotFoundResponse(), @ApiForbiddenResponse(), @ApiUnauthorizedResponse() — erreurs

Pour AuthController :

  • Groupez sous le tag auth
  • login est publique, me nécessite un Bearer token
  • Documentez les codes de réponse 200 et 401

Pour UsersController :

  • Groupez sous users, marqué @ApiBearerAuth('JWT-auth') au niveau de la classe
  • Documentez la restriction ADMIN sur create et delete
  • Documentez les 404 et 403 possibles

Faites de même pour TeamsController, ProjectsController, TasksController.

Étape 4 — Ajouter @ApiProperty sur les DTOs

Voici un exemple pour un champ de CreateUserDto :

import { ApiProperty, ApiPropertyOptional } from '@nestjs/swagger';

export class CreateUserDto {
  @ApiProperty({ example: 'alice@taskflow.dev', description: 'Adresse email unique' })
  @IsEmail({}, { message: 'Adresse email invalide' })
  email: string;

  // TODO: annoter les autres champs avec @ApiProperty ou @ApiPropertyOptional
  // Pour les champs optionnels : @ApiPropertyOptional({ enum: UserRole, default: UserRole.MEMBER })
  // Pour les passwords : { writeOnly: true }
}

Annotez tous les champs de CreateUserDto, CreateTeamDto, CreateProjectDto, CreateTaskDto.

Les UpdateDto héritent automatiquement des annotations via PartialType.

Étape 5 — Tester via Swagger UI

  1. Ouvrir http://localhost:3000/docs
  2. Cliquer "Authorize" → coller le token obtenu via POST /api/auth/login
  3. Tester GET /api/users → réponse en live
  4. Tester POST /api/users avec l'exemple pré-rempli
  5. Tester un cas d'erreur (ID invalide → 404)

Checkpoint ✓

  • http://localhost:3000/docs accessible
  • Tous les endpoints apparaissent groupés par tag
  • Le bouton "Authorize" fonctionne avec le token JWT
  • Les DTOs affichent des exemples dans le formulaire
  • On peut tester GET /api/users directement depuis l'UI
