Étape 2 — Décorateurs utilitaires

Ces décorateurs sont des utilitaires courts — ils vous sont fournis :

src/auth/decorators/public.decorator.ts :

import { SetMetadata } from '@nestjs/common';
export const IS_PUBLIC_KEY = 'isPublic';
export const Public = () => SetMetadata(IS_PUBLIC_KEY, true);

src/auth/decorators/current-user.decorator.ts :

import { createParamDecorator, ExecutionContext } from '@nestjs/common';

export const CurrentUser = createParamDecorator(
  (_data: unknown, ctx: ExecutionContext) => {
    const request = ctx.switchToHttp().getRequest();
    return request.user;
  },
);

src/auth/auth.module.ts — configuration du module :

@Module({
  imports: [
    UsersModule,
    PassportModule,
    JwtModule.registerAsync({
      imports: [ConfigModule],
      inject: [ConfigService],
      useFactory: (config: ConfigService) => ({
        secret: config.get<string>('JWT_SECRET'),
        signOptions: { expiresIn: config.get<string>('JWT_EXPIRES_IN', '24h') },
      }),
    }),
  ],
  controllers: [AuthController],
  providers: [AuthService, LocalStrategy, JwtStrategy],
  exports: [JwtModule],
})
export class AuthModule {}

Étape 7 — Guard global dans AppModule

Enregistrer JwtAuthGuard comme guard global (s'applique à toutes les routes) :

// app.module.ts
import { APP_GUARD } from '@nestjs/core';

providers: [
  {
    provide: APP_GUARD,
    useClass: JwtAuthGuard,
  },
],

Importer AuthModule dans les imports.

Ne pas oublier d'ajouter @Public() sur /health si cette route doit rester publique.

Étape 8 — Tests HTTP

Créer src/http/auth.http :

@baseUrl = http://localhost:3000/api
@token = REMPLACER_PAR_LE_TOKEN

### Login ✅
POST {{baseUrl}}/auth/login
Content-Type: application/json

{
  "email": "alice@taskflow.dev",
  "password": "password123"
}

### Login mauvais mot de passe → 401
POST {{baseUrl}}/auth/login
Content-Type: application/json

{
  "email": "alice@taskflow.dev",
  "password": "mauvais"
}

### Me — qui suis-je ?
GET {{baseUrl}}/auth/me
Authorization: Bearer {{token}}

### Route protégée sans token → 401
GET {{baseUrl}}/users

Checkpoint ✓

  • POST /api/auth/login avec bons credentials → { access_token, user }
  • POST /api/auth/login avec mauvais mot de passe → 401
  • GET /api/users sans token → 401
  • GET /api/users avec token → liste des users
  • GET /api/auth/me avec token → user connecté
  • JWT_SECRET dans .env, jamais exposé
