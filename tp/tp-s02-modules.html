TP 02 — Architecture Modulaire & Docker Compose

Durée : 45 minutes Objectif : ajouter Docker Compose + PostgreSQL, ConfigModule, 5 modules métier, et le préfixe /api.

Étape 1 — Docker Compose + PostgreSQL

À la racine du projet taskflow-api, créer docker-compose.yml :

services:
  postgres:
    image: postgres:16-alpine
    container_name: taskflow_postgres
    environment:
      POSTGRES_DB: ${DB_NAME:-taskflow}
      POSTGRES_USER: ${DB_USER:-taskflow}
      POSTGRES_PASSWORD: ${DB_PASSWORD:-taskflow}
    ports:
      - "${DB_PORT:-5432}:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${DB_USER:-taskflow}"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  pgdata:

Démarrer PostgreSQL :

docker compose up -d
docker compose ps
# → taskflow_postgres   running

Étape 2 — Variables d'environnement

Installer @nestjs/config :

npm install @nestjs/config

Créer .env :

NODE_ENV=development

DB_HOST=localhost
DB_PORT=5432
DB_NAME=taskflow
DB_USER=taskflow
DB_PASSWORD=taskflow

Créer .env.example (même structure, valeurs génériques) et vérifier que .env est bien dans .gitignore.

Étape 3 — Enregistrer ConfigModule dans AppModule

Modifier src/app.module.ts pour importer ConfigModule.forRoot() avec les options :

  • isGlobal: true — disponible partout sans ré-import
  • envFilePath: '.env' — chemin du fichier de variables

Consultez la doc : docs.nestjs.com/techniques/configuration

Redémarrer et vérifier qu'il n'y a pas d'erreur.

Étape 4 — Générer les 5 modules métier

nest g resource users --no-spec
nest g resource teams --no-spec
nest g resource projects --no-spec
nest g resource tasks --no-spec
nest g resource comments --no-spec

Choisir REST API pour chaque ressource. L'option --no-spec évite de générer les fichiers de test pour l'instant.

Vérifier que AppModule importe bien les 5 nouveaux modules. La CLI l'a fait automatiquement — inspectez app.module.ts pour vous en assurer.

Étape 5 — Préfixe global /api

Dans src/main.ts, ajouter une ligne après NestFactory.create() pour que toutes les routes soient préfixées par /api :

Méthode : app.setGlobalPrefix('api')

Tester les nouvelles routes :

curl http://localhost:3000/api/users
# → []

curl http://localhost:3000/api/tasks
# → []
